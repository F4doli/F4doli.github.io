Démarche initiale
#

La demande du client était de pouvoir centraliser leur travail, afin d’accéder à leur ressources depuis n’importe quel endroit et quel poste de l’entreprise. Suite à l’audit de mon maître d’apprentissage sur site, il a donc décidé de préparer en atelier un serveur Active Directory et RDS. Les deux machines ont été créées sur l’hyperviseur Hyper-V en tant que machine virtuelle.
Toute la partie préparation a été conduite par mon maître d’apprentissage, j’aborderai donc la partie qui m’a été confié, étant l’installation sur site.

Installation du serveur sur site et migration
#

Ma mission avec celle d’un autre technicien est donc de faire l’installation de ce serveur directement chez le client.
Comme nous devions remplacer un serveur existant, nous avons brancher le nouveau serveur sans intervenir sur l’ancien, afin d’effectuer quelques test et nous assurer que celui-ci était bien paramétré.
Une fois que nous étions certains du bon fonctionnement, nous avons désactivé le service DHCP de l’ancien serveur afin que le nouveau serveur prenne le relai. Le but étant de laisser l’ancien serveur accessible car des données communes étaient nécessaires pour que les clients effectuent leur travail.
Maintenant que le service DHCP du nouveau serveur est opérationnel, les clients ont donc récupéré le DNS da la machine virtuelle Active Directory.

Nous pouvions alors démarrer notre migration. Les étapes qui vont être décrites ci-dessous ont été ralisées sur chaque poste nécessitant une intégration au domaine :

  • Intégration de l’ordinateur au domaine
  • Connexion de la nouvelle session via le protocole RDP sur le serveur RDS
  • Migration des données du client (discussion avec lui afin de savoir ce qu’il souhaite garder ou non)
  • Installation des nouvelles applications (Office & logiciel utilisés par l’utilisateur)
  • Validation du bon fonctionnement avec le client

La partie utilisateur étant terminée, Nous devions maintenant migrer le logiciel métier utilisé par tous les utilisateurs qui disposerait d’un machine virtuelle dédiée sur le serveur, ainsi que les répertoires communs.
Concernant la migration des répertoires communs, nous avons utilisé un outil intégré dans les distributions Windows et se nommant Robocopy.
La volumétrie des données étant importante et l’ancien serveur disposant d’un vieux disque dur, l’importation fût très longue, nous l’avions laissé tourné la nuit.

Concernant la migration du logiciel métier, nous avons directement appeller l’éditeur du logiciel. Il s’est occupé de la partie installation et migration de la base de données.

Une fois que tout fût opérationnel, nous avons donc débrancher et enlever l’ancien serveur.

Cette migration a durée deux jours, à deux techniciens.

Compétences déployées
#

Avec un projet comme celui-ci, j’ai pu acéquerir et consolider les compétences suivantes :

  • Mettre en place et vérifier les niveaux d’habilitation associéz à un service
  • Traiter des demandes concernant les services réseau et système, applicatifs
  • Analyser les objectifs et les modalités d’organisation d’un projet
  • Accompagner les utilisateurs dans la mise en place d’un service
